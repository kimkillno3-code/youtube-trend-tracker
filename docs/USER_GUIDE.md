# YT 토픽 파인더 — 사용자 가이드

YouTube 트렌드를 수집하고 분석하여 콘텐츠 기회를 발견하는 대시보드입니다.

---

## 1. 시작하기

### 1-1. 사전 준비

| 항목 | 설명 |
|------|------|
| **Python** | 3.9 이상 |
| **YouTube API 키** | [Google Cloud Console](https://console.cloud.google.com/apis/credentials)에서 발급 |
| **YouTube Data API v3** | Google Cloud Console > API 및 서비스 > 라이브러리에서 활성화 |

### 1-2. API 키 설정

프로젝트 루트에 `.env` 파일을 생성하고 아래 내용을 입력하세요:

```
YOUTUBE_API_KEY=발급받은_API_키
```

API 키가 등록되었는지는 **설정** 페이지에서 확인할 수 있습니다 (마스킹 처리되어 표시).

### 1-3. 실행

```bash
pip install -r requirements.txt
streamlit run dashboard/app.py
```

브라우저에서 `http://localhost:8501`로 접속하세요.

## 3. 트렌드 대시보드

### 3-1. 실시간 트렌드 탭

한국 YouTube에서 현재 인기 급상승 중인 영상 30개를 가져옵니다.

**사용 방법:**
1. **실시간 인기 영상 가져오기** 버튼 클릭
2. 수집 완료 후 핵심 지표(수집 영상 수, 평균/최고 조회수, 인기 카테고리) 확인
3. **카테고리 필터**로 관심 있는 카테고리만 필터링
4. 영상 카드 클릭 시 YouTube 영상으로 이동
5. **엑셀 다운로드**로 데이터 저장

> **API 사용량:** ~2 유닛 (일일 한도 10,000 유닛 기준 약 5,000회 가능)

### 3-2. 주간 인기 탭

인기 급상승 영상 50개를 가져와 조회수 기준으로 정렬한 TOP 30을 보여줍니다.

**사용 방법:**
1. **주간 인기 영상 분석** 버튼 클릭
2. 실시간 탭과 동일한 방식으로 결과 확인
3. **실시간 중복 제외** 토글: 실시간 탭과 겹치는 영상을 제외하여 새로운 영상만 확인

> **API 사용량:** ~2 유닛

### 3-3. 분석 리포트 탭

수집된 전체 데이터를 기반으로 트렌드 인사이트를 제공합니다. **API를 사용하지 않습니다.**

**제공 정보:**
- **핵심 지표 요약**: 수집 영상 수, 평균/최고 조회수, 인기 카테고리
- **카테고리별 분포**: 도넛 차트로 카테고리 비중 시각화
- **자주 쓰인 태그 TOP 15**: 어떤 태그가 트렌드에 많이 사용되는지 확인
- **조회수 분포**: 트렌드 영상의 조회수 구간별 분포
- **참여율 TOP 10**: 좋아요/조회수 비율이 높은 영상
- **채널별 등장 횟수**: 트렌드에 자주 등장하는 채널
- **영상 길이 분포**: 어떤 길이의 영상이 트렌드에 많은지
- **AI 인사이트**: 평균 영상 길이 분석, 소규모 채널 성과

## 4. 키워드 검색

특정 키워드로 YouTube 영상을 검색하고, 경쟁 영상을 벤치마킹할 수 있습니다.

**사용 방법:**
1. **검색어** 입력 (예: AI, 먹방, 브이로그)
2. **기간** 선택: 최근 1주일 / 1개월 / 3개월 / 전체
3. **영상 길이** 선택: 전체 / 4분 미만 / 4~20분 / 20분 이상
4. **검색하기** 버튼 클릭
5. 결과 확인:
   - 핵심 지표 (평균 조회수, 최고 조회수, 평균 길이, 소채널 성과)
   - 인사이트 박스 (인기 카테고리, 많이 쓰인 태그, 소채널 분석)
   - **정렬**: 조회수순 / 최신순 / 관련도순
   - **카테고리 필터**: 카테고리별 결과 필터링
   - **엑셀 다운로드** / **썸네일 모아받기**

> **API 사용량:** ~102 유닛 (일일 약 98회 검색 가능)
> 검색은 `search.list` API를 사용하며 1회당 100유닛으로, **가장 비용이 높은 액션**입니다.

## 5. 설정

### 5-1. API 키 상태
- 등록된 API 키를 마스킹하여 표시 (앞 4자리 + *** + 뒤 4자리)
- 미등록 시 설정 방법 안내

### 5-2. API 할당량 (일일)
- **오늘 사용량**: 오늘 사용한 API 유닛 수
- **잔여 할당량**: 남은 유닛 수
- **사용률**: 프로그레스 바로 시각화
- 70% 이상: 안내 메시지
- 90% 이상: 경고 메시지
- 자정(태평양 시간, 한국 시간 오후 4~5시) 초기화

### 5-3. 최근 수집 기록
- 최근 10건의 수집 기록을 확인할 수 있습니다
- 각 기록: 수집 유형, 시작 시간, 수집 영상 수, 할당량 사용량, 상태, 에러 메시지

## 6. 카테고리 필터 사용법

실시간 트렌드, 주간 인기, 키워드 검색 결과에서 카테고리 필터를 사용할 수 있습니다.

| 기능 | 설명 |
|------|------|
| **카테고리 클릭** | 해당 카테고리의 영상만 표시 (복수 선택 가능) |
| **다시 클릭** | 선택 해제 |

선택된 카테고리는 하이라이트로 표시됩니다. YouTube 표준 카테고리(엔터테인먼트, 게임, 음악 등) 기준으로 분류됩니다.

## 7. 성과 별점 기준

영상 카드에 표시되는 별점은 **구독자 대비 조회수 비율**로 산출됩니다.

| 별점 | 기준 (조회수 / 구독자) | 의미 |
|------|----------------------|------|
| ★★★★★ | 50배 이상 | 초대박 |
| ★★★★ | 10배 이상 | 대박 |
| ★★★ | 2배 이상 | 우수 |
| ★★ | 0.5배 이상 | 보통 |
| ★ | 그 외 | 기본 |

> **활용 팁:** 구독자 5만 이하 소규모 채널이 높은 별점을 받은 영상이 있다면, 해당 키워드/주제가 **블루오션**일 가능성이 높습니다.

## 8. API 할당량 관리

YouTube Data API v3의 일일 무료 할당량은 **10,000 유닛**입니다.

### 액션별 비용

| 액션 | 사용 유닛 | 하루 가능 횟수 | 비용 요인 |
|------|----------|--------------|----------|
| 실시간 인기 영상 가져오기 | ~2 | ~5,000회 | `videos.list` + `channels.list` |
| 주간 인기 영상 분석 | ~2 | ~5,000회 | `videos.list` + `channels.list` |
| 키워드 검색 | ~102 | ~98회 | `search.list`(100) + `videos.list`(1) + `channels.list`(1) |

### API를 사용하지 않는 기능

다음 기능들은 이미 수집된 로컬 데이터를 사용하므로 **할당량 소모가 없습니다:**

- 분석 리포트 탭 (차트, 인사이트)
- 카테고리 필터링
- 정렬 변경
- 엑셀 다운로드
- 썸네일 모아받기 (YouTube CDN 직접 접근, API 아님)
- 설정 페이지 전체
- 라이트/다크 모드 전환

### 할당량 절약 팁

1. **검색은 신중하게**: 1회 검색에 ~102유닛. 하루 98회까지 가능
2. **트렌드 수집은 부담 없이**: 1회에 ~2유닛으로 매우 저렴
3. **분석 리포트 적극 활용**: 한 번 수집한 데이터는 무제한 분석 가능
4. **설정 페이지에서 사용량 확인**: 할당량 70% 이상 사용 시 안내 메시지 표시

## 9. 에러 대응

| 에러 메시지 | 원인 | 해결 방법 |
|------------|------|----------|
| API 키가 설정되지 않았어요 | `.env` 파일 미생성 또는 키 누락 | 설정 페이지의 안내에 따라 `.env` 파일 생성 |
| 일일 할당량을 초과했어요 | 10,000 유닛 소진 | 태평양 시간 자정(한국 시간 오후 4~5시) 이후 자동 초기화 |
| API 키 권한이 부족해요 | YouTube Data API v3 미활성화 | Google Cloud Console에서 API 활성화 확인 |
| 네트워크 연결을 확인해주세요 | 인터넷 연결 문제 | 네트워크 상태 확인 후 재시도 |

## 10. 데이터 저장 구조

| 항목 | 위치 |
|------|------|
| 데이터베이스 | `data/youtube_trends.db` (SQLite) |
| 환경 설정 | `.env` (프로젝트 루트) |
| 테마 설정 | `.streamlit/config.toml` |

수집된 모든 데이터는 로컬 SQLite에 저장되며, 서버나 외부에 전송되지 않습니다.
